# Git Branches

Bei der Projektarbeit, insbesondere im Team, ist es wichtig, unabhängig voneinander an Funktionen zu arbeiten, damit diese die Arbeit anderer nicht beeinträchtigen. Git bietet uns **Branches**, um unsere aktuelle Arbeit bis zur Fertigstellung vom gemeinsamen Code des Teams zu trennen.

Ein Branch ermöglicht es, sich vom Hauptentwicklungszweig abzuspalten. Der neue Branch teilt einen Teil seiner Commit-Historie mit dem Hauptzweig. Bei einem bestimmten Commit zweigt der neue Branch ab, und die Commit-Historien unterscheiden sich.
